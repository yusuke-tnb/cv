## 要約

- ２社の企業でCTOとしての経験があり、ビジネスとエンジニアリングの架け橋となれます。ビジネス要望を技術要件に落とし込み、エンジニアを主導して開発を進めていく動きが可能です。
- 新規事業立ち上げの経験があり、それに伴う技術選定やアーキテクチャ設計、開発組織の立ち上げ、IT人材の採用など、エンジニアリング周りは一通り経験してきました。
- コンセプチュアルスキルを活かして全体目標や解決すべき経営課題から具体的行動に落とし込み、主体的に行動します。
- コードを書くのが好きで、なるべく時間を見つけてコーディングの時間を取るようにしています。

## 行動指針
**【其の一】** 顧客体験と開発者体験の双方を向上させることで、質の高い最高のプロダクトを生み出します.　　

**【其の二】** プロダクトとチームの両面からマネジメントし、エンジニアが働きやすい環境を整えます。

**【其の三】** 経営的な観点からエンジニアリングを支えます。そのために「効果を定量的に測定できる環境の整備」を意識します。　　　　

## スキルセット
- 技術的にはフロントエンドも触れるバックエンドエンジニアという立ち位置となります。、
- インフラはAWSのVPC/ECS/CloudFront/S3/RDS/EC2/Dynamo/ElastiCache/SQSなどの基本サービスであれば構築可能です。
